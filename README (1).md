## `02` &nbsp; Projects

<br/>

### ◆ &nbsp; Comparative Sentiment Analysis — Sparse vs Dense Embeddings

```
TYPE  ──  NLP · Embedding Benchmarking · Deep Error Analysis
DATA  ──  50,000 IMDB reviews
SCORE ──  89.3% accuracy (Word2Vec/FastText + LSTM)
```

> Two paradigms. One rigorous comparison. Not just "which is better" — but *where* and *why*.

| Approach | Pipeline | Accuracy |
|---|---|:---:|
| Sparse | TF-IDF → Logistic Regression | baseline |
| Dense | Word2Vec / FastText → LSTM | **89.3%** |

**What separated this from a standard project:**

- Used **t-SNE** to visualize embedding cluster separability — validated representational quality, not just downstream accuracy
- Deep error analysis on **negation** (`not good`), **contrastive conjunctions** (`good but...`), and **mixed-sentiment** reviews
- Mapped *where* each approach systematically fails — structural analysis, not anecdote

`Python` · `Scikit-learn` · `Gensim` · `TensorFlow/Keras` · `Matplotlib`

### ◆ &nbsp; Autism Spectrum Disorder Prediction

```
TYPE  ──  Classification · Imbalanced Learning · Clinical Data
FOCUS ──  Reliability over raw accuracy
```

> In clinical ML, false negatives aren't a metric — they're a consequence. Built accordingly.

- Applied **SMOTE** for class imbalance — recall tuned deliberately over precision
- **Random Forest + XGBoost** with cross-validated hyperparameter search
- Evaluated via precision-recall curves, not just accuracy scores

`Python` · `Scikit-learn` · `XGBoost` · `imbalanced-learn`
